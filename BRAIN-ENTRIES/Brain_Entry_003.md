## Comet Browser — Lessons Learned (Critical for Future Use)

### The Core Problem
Sessions 1–6 included significant time wasted in visual/creative apps via browser
automation. This was identified mid-thread and explicitly called out by the user:
*"control browser in creative visual apps like Canva, Shopify or Photopea are agonizing
to watch, lots of waste."*

### Comet Browser: DO Use For
| Task Type | Why It Works |
|---|---|
| Reading/writing Google Docs & Sheets | Text-based, reliable, fast |
| Shopify settings forms (text fields, dropdowns, toggles) | Structured UI, predictable |
| Navigation menus, legal policies, SEO fields | Form inputs, no visual guesswork |
| Web research and competitive analysis | Search + read, no fragile coordinates |
| Content generation pasted into forms | Pure text output |
| Filling structured data (hit lists, changelogs) | High accuracy, low waste |

### Comet Browser: DO NOT Use For
| Task Type | Why It Fails |
|---|---|
| Canva — logo/graphic design | Drag handles, color pickers, floating panels = coordinate hell |
| Photopea / any pixel editor | Same problem; scripts hang, layers misbehave |
| Shopify Theme Editor (visual) | iframe-heavy, visual-only feedback loops |
| Any tool requiring precise mouse artistry | Fragile, slow, requires constant correction |
| Rapid iterative visual work | Too many round-trips to screenshot → click → verify |

### Better Framework for Visual Asset Work
1. **You create assets yourself** in Canva/Photopea/Figma on desktop — faster, no waste
2. **Comet handles everything upstream and downstream**: brief writing, copy for the
   asset, specs, then uploading the finished file to Shopify/GBP/wherever
3. **For branding specs**: Comet writes the design brief (colors, fonts, dimensions,
   copy) → you execute in the visual tool → Comet uploads + deploys
4. **Exception**: Photopea scripting via the browser console *can* work for
   non-visual, programmatic tasks (fill a layer with hex color, resize canvas) —
   but only if the script is simple and pre-tested. Abandon if it hangs once.

## Workflow Model — What Good Comet Sessions Look Like

Based on this thread, the pattern that worked vs. didn't:

### High-ROI Comet Tasks (do these)
- Read changelog → understand current state → act
- Update Shopify text settings: SEO, policies, navigation, page content
- Write and paste content into Google Docs
- Research competitors, keywords, local businesses
- Populate spreadsheets (hit lists, trackers)
- Generate templates, SOPs, email sequences
- Blog post drafts → paste into Shopify Pages

### Low-ROI Comet Tasks (avoid or minimize)
- Anything requiring clicking inside a canvas or design surface
- Shopify theme visual editor (customize section layouts, colors via sliders)
- Any tool with heavy JavaScript rendering that breaks coordinate targeting

### Session Structure That Works
1. Start by reading the changelog (Google Doc) to get current state
2. Identify the 1-3 highest-value text/data tasks
3. Execute those — no detours into visual tools
4. End every session by updating the changelog
5. If a visual task comes up: write the spec/brief, park it for the user to execute
