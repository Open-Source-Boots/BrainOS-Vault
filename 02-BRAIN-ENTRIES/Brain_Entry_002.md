## 3. Key Lesson — AI Browser Automation Boundaries

This is important enough to encode as a standing rule:

> **AI browser automation works best with tight, scoped, reversible tasks and
> frequent human checkpoints. Giving the AI an open-ended "get as far as you
> can" instruction in a browser session leads to compounding decisions,
> aesthetic drift, and outputs that don't reflect the human's actual vision.**

**What went wrong in the March session:**
- Too much autonomy granted over visual/design decisions
- Changes stacked on changes without approval loops
- End result didn't match intent and couldn't be easily unwound
- Time cost exceeded value produced

**Standing rule for future sessions:**
- Define a single scoped task before starting browser control
- Set a checkpoint after every 2–3 actions ("does this look right?")
- Prefer reversible actions; flag irreversible ones before executing
- If a session feels like it's drifting, stop and reassess — don't keep going
